7/7 ifやforなどにカッコがいらなかった件

七夕ですね。この記事はGPT-4oによって生成されています。AIの記事も校閲はしてます。修正もします。

最近、mdn読んでてて気づいたんですよ。それは、ifやforなどの制御構文で、一行だけのものに関してはカッコなしでも文法的に正しいということです。

例えば、JavaScriptで以下のようなコードを書いたことがある人は多いでしょう。

if (condition) {
    doSomething();
}

このように、一般的には条件式の後にカッコを使うのが普通です。しかし、実はカッコ内が一行であるならば例えば、以下のように書いても同じ結果が得られます。

if (condition) doSomething();

このように、一行だけの処理に関しては、カッコを省略しても問題ないんです。

カッコを使わないことで、コードがより簡潔に見えることがあります。ただし、複数行の処理を行う場合や、コードの可読性を重視する場合には、やはりカッコを使った方が良いでしょう。

カッコを省略するかどうかは、状況に応じて使い分けると良いですね。
